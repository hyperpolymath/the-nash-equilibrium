= Trade-Card Library (eXchange Pillar)
:toc: preamble
:icons: font
:sectnums:
:sectnumlevels: 2

== Purpose

The eXchange pillar is implemented through a small library of *typed* trade
cards. Every agreement at the table is a card — there is no free-form
bartering — and every card carries: parties, terms, duration, defection
clauses, and a printed mechanical/strategic cue line on the card face. This
document is the canonical card library; printed cards in the box should
match this spec word-for-word in clause text.

Foundations are specified in `economic-foundations.adoc`:

* §2.2 (information symmetry, asymmetric information, fog) — informs which
  cards may be played face-down and what penalty applies on reveal.
* §2.4 (externalities) — informs the externality-related cards (Carbon
  Offset, Collective Emissions Pact).
* §5 (Pigouvian register) — the boundary dashboard against which
  externality clauses register.

The design decision (2026-04-30) is that **cards do not carry historical
flavour text.** The cue line is a mechanical mnemonic or a strategic prompt
to help the player reason about the card in play. Historical and
educational context lives in the rulebook appendix and the educational
supplement, not on the card face.

== Card design conventions

Every trade card uses the same template:

----
  ┌─────────────────────────────────────────────────┐
  │ [icon]  CARD NAME                  [Class]      │
  │         [Bilateral / Multilateral / Coercive]   │
  ├─────────────────────────────────────────────────┤
  │ TERMS:    Party A obligations                   │
  │           Party B obligations                   │
  │                                                 │
  │ DURATION: [one-off | N turns | permanent]       │
  │                                                 │
  │ QUORUM:   [N/A | k of n signatories]            │
  │                                                 │
  │ DEFECTION:                                      │
  │           If A breaks: ...                      │
  │           If B breaks: ...                      │
  │                                                 │
  │ LEDGER:   [conditions under which a copy of     │
  │           this card moves to the defection      │
  │           ledger]                               │
  ├─────────────────────────────────────────────────┤
  │ CUE: [mechanical/strategic prompt for play —    │
  │       not flavour]                              │
  └─────────────────────────────────────────────────┘
----

*Class* is one of: Cooperative, Commodity, Procedural, Externality,
Coercive, Post-conflict, Multilateral. The class determines which
discard/ledger pile the card joins on resolution.

*Quorum* applies only to Multilateral cards. Bilateral cards require both
parties to sign.

*Ledger* events make a face-up token visible to all players for the rest
of the game (see `gameplay-storyboard.adoc` Phase 3 design check 3.2).

== Defection-ledger interactions

The defection ledger is a public face-up zone on the main board where
copies of cards land when one of the conditions in their LEDGER row fires.
The ledger has these systemic effects:

. *Future MFN clauses cannot be signed by a player carrying ≥2 ledger
  tokens.* (Card 4 — credibility cost.)
. *Repeated Long-Term Supply Contract defections by the same player block
  further contracts of that type with anyone.* (Card 2 — reputation
  signalling.)
. *Tariff Walls and Reparations are first-class ledger entries* — they
  are inherent records, not consequences of breakage.
. *At end-of-era*, players with the cleanest ledger receive a small
  Stability bonus (+1) and players with the most-cluttered ledger receive
  a Stability penalty (-1). This implements the long-run reputation effect
  the README's repeated-game framing requires.

The ledger is an *information* structure (`economic-foundations.adoc`
§2.2): it is common knowledge once a card lands there, and it shapes the
choice set of all future trades. A player's prior defections cannot be
hidden — but a player's prior compliance is also visible, which is why
the ledger is a strategic asset for cooperators and not only a punishment
register.

== Open design questions

. *Should trades be playable face-down?* `economic-foundations.adoc` §6
  discusses asymmetric-information trades (closed agreements with a
  Stability cost on later reveal). The library above assumes open play;
  closed-play variants need a separate design pass and likely a small
  set of "Lemon" cards (in the Akerlofian sense) that exploit closed
  play.

. *Card-card interactions.* What happens when a Tariff Wall is imposed on
  a player carrying an active Long-Term Supply Contract with the
  imposer? Current draft answer: the LSC suspends for the duration; both
  sides accumulate "debt" tokens that resolve when the wall lifts. This
  needs playtest.

. *Quorum rules for Multilateral cards beyond Card 8.* Are there
  multilateral commodity contracts (e.g., a four-way Labour-pooling
  cooperative)? Likely yes for Socialist-pathway design space, but not
  yet specified.

. *Card cost.* Cards above are assumed *free to play* — players have
  trade cards in hand and can play them with no Capital cost. A future
  iteration may charge a small Capital cost per signing, internalising
  the transaction-cost insight from Coase (1960). This would also gate
  spam.

. *Card supply.* How many copies of each card exist in the game box?
  The Debt-for-Nature Swap could plausibly recur many times in a single
  game; the Reparations card should not be plentiful (it should feel
  like a punctuation event). Print runs need calibration through
  playtesting.

. *Negotiation phase time-boxing.* Phase 3 in the storyboard hits a
  natural time problem: with 8 cards available and 4 players, complex
  multi-step negotiations are possible. Recommend a 5-minute hard cap
  on Phase 3 with an explicit "anyone may pass" early termination.
