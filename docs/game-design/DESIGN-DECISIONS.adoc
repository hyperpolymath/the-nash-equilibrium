== Purpose

This document records every significant design decision as dialogue and FAQ,
preserving the reasoning chain so we can trace the path through development
and learn from it. Each entry captures *what* was decided, *why*, *who* raised
it, *what alternatives existed*, and *what we'd revisit if assumptions change*.

Updated continuously as the project evolves.

'''

== Session 1: Project Genesis (2026-04-04)

=== DD-001: Game Name — "The Nash Equilibrium"

*Raised by:* Jonathan

*Decision:* Working title is *The Nash Equilibrium*.

*Reasoning:* The name directly references John Nash's equilibrium concept from
non-cooperative game theory, which is also the core meta-strategic mechanic — the
winning position is the one where no player can improve by unilaterally changing
strategy. The name signals the game's intellectual depth and educational angle.

*Alternatives considered:* "Econosphere" was the subtitle in the initial design
document but serves better as a descriptor than a title.

*Status:* ACTIVE (working title, may change before Kickstarter)

*FAQ:*

* _Q: Won't people confuse it with a maths textbook?_
  A: The game theory angle is a differentiator in the board game market, not a
  liability. Terraforming Mars doesn't shy from its science. The hook is
  "What if you could play through human history — and change the ending?"
  The Nash Equilibrium is the *why*, not the *what*.

* _Q: Any trademark concerns with "Nash Equilibrium"?_
  A: The term is a widely-used mathematical concept, not a trademark. John Nash
  is deceased (2015). The phrase is in the public domain as a scientific term.
  No issues expected.

=== DD-002: Genre — 6X (not 4X)

*Raised by:* Jonathan

*Decision:* The game is a *6X* strategy game, extending the classic 4X
(eXplore, eXpand, eXploit, eXterminate) with eXchange and eXperiment.

*Reasoning:*

* *eXchange* — trade, diplomacy, and game-theoretic negotiation are first-class
  strategic pillars, not afterthoughts. In most 4X games, diplomacy is a thin
  veneer over military dominance. Here, trade agreements and alliances are modelled
  as repeated games where cooperation and betrayal have real mechanical consequences.
  This is what makes the "Nash Equilibrium" name load-bearing.
* *eXperiment* — speculative research with branching non-deterministic outcomes
  and asymmetric payoffs. Creates information asymmetry that feeds back into
  eXchange. Traditional tech trees are linear and deterministic — boring.

*Alternatives considered:* Staying with 4X and making trade/research sub-mechanics.
Rejected because the whole point is elevating these to first-class pillars.

*Status:* RESOLVED

*FAQ:*

* _Q: Is "6X" a recognised genre term?_
  A: Not yet — this game would coin it. That's a marketing advantage ("the
  world's first 6X board game") and a minor risk (explaining a new term).

* _Q: Does adding two pillars make the game too complex?_
  A: The pillars aren't additional complexity — they're additional *structure*.
  Trade and research exist in every strategy game; calling them X's means we
  design them as deeply as combat and exploration. The complexity risk is
  real but separate (see DD-008).

=== DD-003: License — AGPL-3.0-or-later

*Raised by:* Jonathan

*Decision:* Licensed under AGPL-3.0-or-later.

*Reasoning:* This is a father-son co-development project, same pattern as
IDApTIK and Airborne Submarine Squadron. AGPL is the standing policy for
projects co-developed with Joshua. PMPL-licensed tools (Hypatia, panic-attack,
a2ml, etc.) are used as dependencies under their own license — standard
third-party license respect.

*Alternatives considered:* MPL-2.0 (standard for solo hyperpolymath
work). Rejected — co-development with Joshua follows the AGPL precedent.

*Status:* RESOLVED

*FAQ:*

* _Q: Does AGPL affect the physical board game?_
  A: AGPL covers the software components (Tabletop Simulator mod, any digital
  tools, game logic code). The physical board game rules, art, and text are
  covered by the AGPL as creative works distributed with the software. If
  a purely physical edition is released without software, a separate license
  for art/rules could be considered, but AGPL is fine for now.

* _Q: Can PMPL tools be used in an AGPL project?_
  A: Yes. PMPL tools are dependencies, not part of this project's codebase.
  They retain their own license. Standard practice.

=== DD-006: Player Roles — Named Characters vs. Archetypes

*Raised by:* Claude

*Decision:* RESOLVED (2026-07-16, Jonathan with Joshua concurring) — *neither
horn*. Ship an *immutable archetypal Core* plus a *character builder*. The player
supplies the name at their own table; nothing is printed.

*The question (as originally framed):* The initial design uses named
historical/contemporary figures (Queen Isabella, Gandhi, Elon Musk, Green
Activist). Should these be replaced with archetypes (The Colonist, The Pacifist,
The Innovator, The Steward)?

*Why the original framing was wrong:* Both horns lose something. Named figures
carry recognition but also likeness exposure and a shelf-life problem. Flat
archetypes are safe but bland. And the earlier "best of both worlds" compromise
— archetype mechanics with named figures as printed flavour text — still *ships
the name*, so it keeps the exposure it was trying to avoid while gaining little.

*What was decided instead:* A character has layers.

[cols="1,3", options="header"]
|===
| Layer | Content

| *Core* (immutable)
| The archetypal essence — a *type*, never a person. Fixed. Makes the character
  legible at the table. Always *public* (see DD-013).

| *Aspects* (chosen or drawn)
| A small pool that inflects the Core — strengths, a weakness, and an
  appeal/antipathy pair (who rallies to them, who recoils). *Hidden* by default.

| *Name* (free)
| Player-supplied, at the table. Never printed.

| *Private objective* (hidden)
| A quiet parallel goal running alongside the main game. Non-critical, but adds
  challenge and risk-appetite variance.
|===

*Reasoning:*

* *Legal.* The shipped artefact never names or depicts a living individual. It
  ships a *class* of figure — "a Mars-fixated, cult-followed plutocrat" — which is
  satire of a type. Any resemblance is supplied by the players at their own table:
  their speech, not the publisher's.
* *Humour.* Freeing the name is what *licenses* the comedy rather than suppressing
  it. The players do the impression; we hand them the stage. This is an
  opportunity for contemporary satire, not a constraint on it.
* *Ageing.* A type outlives a person. The Core still reads in 2046 when today's
  figures are footnotes.
* *Replayability.* The aspect pool means the same Core plays differently every
  game. A static role card never does.
* *Theme.* Hidden aspects and private payoffs make this a game of *incomplete
  information* — see DD-013. This is the largest gain: the character system
  upgrades the equilibrium the game actually explores.
* *Complexity dial.* The builder doubles as the answer to DD-008 — presets for
  beginners, builder for advanced play. Same system, two depths.

*The existing four roles are not lost — they become presets.* Queen Isabella,
Gandhi, Elon Musk and Green Activist are re-cast as *sample fully-built
characters*: identical mechanics, Core and aspects pre-chosen, name offered as a
*suggestion at the table* rather than a printing. See `player-roles.adoc` for the
framework and the four Cores.

*Design rule — private objectives must tug against the commons.* A private
objective is only interesting when it tempts the player to defect from the shared
planetary-boundaries constraint (DD-007) on a given turn: _help the boundary, or
quietly advance my own escape hatch?_ An objective merely *orthogonal* to the
commons is a solitaire task that pulls attention off the table. An objective
*incentive-misaligned* with it deepens the exact dilemma the game is about.
Misaligned, never independent.

*Alternatives considered:*

* *Named figures as shipped* (the original) — rejected: likeness exposure, ages
  badly, and polarises.
* *Flat archetypes* (The Colonist, The Pacifist…) — rejected: safe but bland;
  loses the recognition hook entirely.
* *Archetype mechanics + named flavour text* (the earlier compromise) — rejected:
  still prints the name, so it retains the exposure while gaining little.

*Status:* RESOLVED — framework. Aspect pool and private-objective deck are Phase 1
drafting work.

*FAQ:*

* _Q: Are there legal issues with using Elon Musk's name?_
  A: Moot under this decision — we don't use it. We ship the *type*; the player
  names their own. The line to hold is *archetype, not portrait*: keep the printed
  Core free of uniquely-identifying specifics (real company names, real quotes,
  one-to-one biography) that would make it identifiable as one particular real
  individual. A Mars-fixated plutocrat with a cult following is a *category*.
  (Not legal advice — but that is the standard satire seam.)

* _Q: What about Gandhi?_
  A: Same resolution, and it defuses the cultural-sensitivity concern too. We ship
  "nonviolent mass-mobiliser, cohesion over growth" as a Core. A table that wants
  to name theirs Gandhi may do so; we are not reducing a real historical figure to
  a stat block *in print*.

* _Q: But players will just name it Elon anyway — isn't that the same thing?_
  A: No, and that distinction is the whole point. The game makes no claim about
  any real person; it hands players a caricature of a *type* and they do as they
  like with it. Fiction is not a documentary. As Jonathan put it: _"The Day of the
  Triffids is never held to a standard of validity, reliability, and fidelity."_

* _Q: Doesn't a builder make setup slower?_
  A: Only if you use it. Presets exist precisely so a Short Game starts in a
  minute — deal a built character and play. The builder is opt-in (DD-008).

=== DD-007: Planetary Boundaries — Dashboard as Shared Constraint

*Raised by:* Design document

*Decision:* The planetary boundaries dashboard is a *shared, visible* constraint
that all players can see and affect. Loss condition: 4+ boundaries at Black (10).

*Reasoning:* The dashboard is the game's central innovation — it makes the
environmental cost of economic activity *visible and mechanical*. Hiding it
would undermine the educational purpose. Making it shared creates the Tragedy
of the Commons dynamic: every player contributes to boundary stress, but no
single player bears the full cost.

*Alternatives considered:*

* Per-player boundary tracking — rejected, loses the shared commons dynamic
* Hidden boundary levels — rejected, loses educational value and creates
  unfair information asymmetry
* Fewer boundaries (e.g., 4-5 instead of 9) — still open for "beginner mode"

*Status:* RESOLVED

=== DD-008: Complexity Management

*Raised by:* Claude

*Decision:* OPEN — needs playtesting data.

*The concern:* The game has 9 boundaries, 5 resources, 4 pathways, 7 turn
phases, 4 eras, 78+ crisis cards, and asymmetric roles. This is *heavy* —
comparable to Terraforming Mars (BGG weight 3.2/5) or heavier.

*Proposed mitigations:*

* *Beginner mode:* 4 boundaries, 3 resources, 1 era (Industrial only), symmetric roles
* *Layered rulebook:* Teach mechanics incrementally across eras
* *Quick-start scenario:* 30-minute single-era game
* *Reference cards:* Per-player aids with phase sequence and resource summary
* *Turn count:* Consider 20-24 turns instead of 32 (see DD-009)
* *Character presets vs. builder:* the DD-006 character builder *is* a complexity
  dial — deal a pre-built preset for beginner mode, expose the aspect pool and
  private objectives for advanced play. Same system, two depths, no separate
  ruleset to maintain.

*Partial resolution (2026-07-16):* the *character-layer* dial is settled by
DD-006. The remaining question — boundary/resource/era counts, and the target
weight — still needs playtesting.

*Status:* OPEN (design question for playtesting)

*FAQ:*

* _Q: Is complexity a dealbreaker?_
  A: Not inherently. Terraforming Mars, Spirit Island, and Twilight Imperium
  are all complex and commercially successful. The key is *approachability* —
  can a new player learn by playing, or must they read 60 pages first?

* _Q: What's the target BGG weight?_
  A: 3.0-3.5 (medium-heavy). Above Catan (2.3), comparable to Terraforming Mars
  (3.2), below Twilight Imperium (4.4).

=== DD-009: Game Length — 32 Turns

*Raised by:* Claude

*Decision:* OPEN — needs playtesting.

*The concern:* 32 turns (8 per era) targeting 90-120 minutes is ambitious. At
3-4 minutes per turn (with trading and crisis resolution), that's 96-128 minutes
in ideal conditions. Real games will run longer with analysis paralysis and
negotiation.

*Options:*

* 32 turns, accept 2-3 hour games (Twilight Imperium audience)
* 24 turns (6 per era), tighter 90-minute target
* Variable era length based on player count and progression triggers
* "Short game" variant with 2 eras instead of 4

*Status:* OPEN (playtesting will determine)

=== DD-010: Cascade Mechanic Threshold

*Raised by:* Design document

*Decision:* When a boundary reaches stress 8+, linked boundaries gain +1 stress
per turn automatically.

*Reasoning:* Models real-world tipping point cascades (Arctic ice -> albedo
feedback, Amazon dieback -> carbon release). Creates urgency and nonlinear
dynamics — the game accelerates toward crisis if boundaries are neglected.

*Risk:* If the threshold is too low, cascades dominate and the game feels
unfair. If too high, cascades never trigger and the mechanic is irrelevant.

*Status:* PROVISIONAL (threshold of 8 needs playtesting validation; may adjust
to 7 or 9)

=== DD-011: The eXchange Pillar — Repeated Game Mechanics

*Raised by:* Design document

*Decision:* Trade and diplomacy modelled as repeated games with mechanical
consequences for cooperation and defection.

*Reasoning:* This is where the Nash Equilibrium name becomes load-bearing.
Traditional board game trading is ad-hoc negotiation. Here, agreements are
*binding* for a number of turns, and breaking them has explicit penalties
(Stability loss, trade route destruction). This creates the iterated Prisoner's
Dilemma dynamic — cooperation builds trust and mutual benefit, defection yields
short-term gain but long-term isolation.

*Open questions:*

* How binding are trade agreements? (1 turn? 3 turns? Until broken?)
* What's the penalty for defection? (Stability loss? Resource penalty? Both?)
* Can agreements be renegotiated without penalty?
* Are agreements public or private?

*Status:* OPEN (core mechanic design needed)

=== DD-012: The eXperiment Pillar — Non-Deterministic Research

*Raised by:* Design document

*Decision:* Research paths branch non-deterministically. Players invest in
speculative technologies with unknown payoffs.

*Reasoning:* Traditional tech trees are deterministic and linear — once you
know the optimal path, research becomes administrative. Non-deterministic
research creates genuine uncertainty and information asymmetry. Players don't
know exactly what they'll get, which feeds into the eXchange pillar (trading
research results, bluffing about capabilities).

*Open questions:*

* How random is "non-deterministic"? (Draw from a deck? Roll dice? Hidden
  research cards revealed on completion?)
* Can players influence outcomes (spend extra Knowledge for better odds)?
* Is research public (everyone sees what you're working on) or private
  (information advantage)?

*Status:* OPEN (core mechanic design needed)

== Session 2: The Character Model (2026-07-16)

=== DD-013: Information Model — Public Core, Hidden Aspects (Bayes–Nash)

*Raised by:* Claude (structure), Jonathan (adopted; Joshua concurring)

*Decision:* RESOLVED — the *Core is always public*; *aspects and the private
objective are hidden*. This makes the game one of *incomplete information*.

*Reasoning:*

Once players hold hidden aspects and private payoffs, nobody knows anyone else's
payoff function — only a distribution over the possibilities. That is precisely a
*Bayesian game*, and its solution concept is the *Bayes–Nash equilibrium*
(Harsanyi's generalisation of Nash to hidden "types"). The *reveal dial* — how
much of your build you disclose, and when — is a *signalling game* (Spence):
disclosure becomes a strategic, sometimes costly, move rather than bookkeeping.

This matters because it makes the character system *load-bearing on the theme*
rather than cosmetic. The game's central concept is upgraded from
complete-information Nash to Bayes–Nash — a richer equilibrium space — and it
does so through the *characters*, which is exactly where players' attention
already is. The players need never hear the word "Bayesian"; they will simply
find they are reading each other rather than reading a stat block.

*Why the Core stays public:* fully hidden information stalls a kitchen-table
game — analysis paralysis sets in, and table talk breaks down when nobody can
read anybody. Public Cores keep the table legible ("everyone can see you're a
techno-plutocrat") while the hidden layer preserves the Bayesian structure.
Legibility on the outside, uncertainty underneath. The eXchange pillar (DD-011)
depends on this: players must be able to read each other *enough to bargain at
all*.

*Alternatives considered:*

* *Everything public* (the current design) — rejected: complete information, no
  signalling, characters reduce to stat blocks.
* *Everything hidden* — rejected: unreadable table, kills negotiation, breaks
  DD-011.

*Status:* RESOLVED (in principle). The exact reveal mechanism — voluntary,
triggered, or costed — is Phase 1 / playtesting work.

[NOTE]
====
*Game title deliberately parked.* The shift to Bayes–Nash raises an obvious
question about the title. It is *explicitly out of scope for now* (Jonathan,
2026-07-16): "The Nash Equilibrium" has the better ring, the audience is not
mathematicians, and the name is — remarkably — unclaimed. DD-001 stands
unchanged. Recorded here only so the reasoning chain survives and this is not
re-litigated by accident.
====

*FAQ:*

* _Q: If the equilibrium is really Bayes–Nash, is the game misnamed?_
  A: No. A Bayes–Nash equilibrium *is* a Nash equilibrium — Nash's concept
  extended to incomplete information, not a rival to it. The title remains
  accurate. See the parking note above; this is not a live question.

* _Q: Doesn't hidden information make the game feel unfair, like DD-007 warned?_
  A: DD-007 rejected hidden *boundary levels* — the shared constraint must stay
  visible, and it does. Hiding a player's own aspects is different: it is
  symmetric (everyone has hidden aspects), it is about *players*, not the world,
  and it is the source of the negotiation the game is built on.
