== Overview

Physical and digital component specifications for The Nash Equilibrium.

== Board Layout

=== Main Board

The main board has three zones:

. *Territory Map* (centre) — hex-based map for eXplore and eXpand
. *Planetary Boundaries Dashboard* (top) — nine boundary tracks visible to all
. *Era Track* (bottom) — four-era progression with turn counter

=== Territory Map

* Hex grid (7x9 recommended for 4 players)
* Terrain types: Ocean, Forest, Plains, Mountains, Desert, Tundra
* Resource nodes on hexes (Labour, Capital, Nature, Energy)
* Trade route paths between player territories
* Settlement sites marked at hex intersections

=== Planetary Boundaries Dashboard

* Nine horizontal tracks (0-10 scale)
* Colour-coded zones: Green (0-3), Amber (4-6), Red (7-9), Black (10)
* Cascade arrows connecting linked boundaries
* Sliding markers for current stress levels
* Global stress indicator (sum display)

=== Era Track

* Four era zones with turn markers (8 turns per era, 32 total)
* Era transition triggers displayed
* Key milestones marked

== Player Boards

Each player has an individual board with:

* Resource pools (Labour, Capital, Nature, Knowledge, Energy)
* Infrastructure slots (factories, farms, labs, trade posts)
* Pathway indicator (Capitalist/Socialist/Green/Techno-Utopian)
* Role card slot with unique bonuses/constraints
* Action tracker (2-3 actions per turn)

== Digital Prototype (Tabletop Simulator)

=== Implementation Priority

. Planetary boundaries dashboard (scripted stress tracking)
. Territory hex map (snap-to-grid placement)
. Card decks (shuffled, era-separated)
. Resource token pools (counter objects)
. Turn structure automation (phase prompts)
. Player boards (individual zones)

=== Scripting Requirements

* Automatic boundary cascade calculations
* Turn phase progression
* Victory/loss condition checking
* Resource generation per turn (based on infrastructure)
* Crisis card resolution prompts

== Art Direction

* *Style:* Clean, information-rich, educational — inspired by Terraforming Mars
  and Wingspan aesthetic
* *Colour palette:* Earth tones with vivid boundary indicators
* *Iconography:* Consistent resource and boundary icons throughout
* *Accessibility:* Colour-blind-safe palettes; text labels on all colour-coded elements
